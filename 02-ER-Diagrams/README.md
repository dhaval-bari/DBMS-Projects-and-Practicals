# ER Diagrams

This folder contains Entity-Relationship (ER) diagram case studies based on real-world database scenarios.

These case studies demonstrate how to design databases using entities, attributes, relationships, and constraints.

---

## Case Studies Included

### 1. University Database
- Manages students, courses, instructors, and departments
- Includes weak entities (SECTION)
- Handles many-to-many relationships (ENROLLMENT)

[View Case Study](University-Database)

---

### 2. Hospital Database
- Models patients, doctors, wards, and treatments
- Includes associative entities (ADMISSION, TREATMENT)
- Demonstrates complex real-world relationships

[View Case Study](Hospital-Database)

---

### 3. E-Commerce Database
- Simulates an online shopping platform
- Includes product catalog, orders, payments, and vendors
- Demonstrates:
  - Self-referencing relationships (CATEGORY)
  - Weak entities (ORDER_ITEM, ADDRESS)
  - Multiple many-to-many relationships

[View Case Study](E-Commerce-Database)

---

## Concepts Covered

- Entities & Attributes
- Relationships (1:1, 1:M, M:N)
- Weak Entities
- Multivalued Attributes
- Associative Entities
- ER → Relational Mapping

---

## Purpose

The goal of this section is to:
- Understand database design at a conceptual level
- Convert ER diagrams into relational schemas
- Apply DBMS concepts to real-world systems
